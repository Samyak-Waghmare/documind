1. OVERVIEW

CloudSync REST API provides programmatic access to cloud storage and synchronization
services. This document specifies the complete API surface for version 3.x.

Base URL: https://api.cloudsync.io/v3
Authentication: OAuth 2.0 Bearer Token + API Key

2. AUTHENTICATION

2.1 OAuth 2.0 Flow:
POST /oauth/token
Request:
{
  "grant_type": "client_credentials",
  "client_id": "YOUR_CLIENT_ID",
  "client_secret": "YOUR_CLIENT_SECRET",
  "scope": "read write delete"
}

Response:
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "scope": "read write delete"
}

3. FILE OPERATIONS

3.1 Upload File
POST /files
Headers: Authorization: Bearer {token}, Content-Type: multipart/form-data
Body: file (binary), folder_id (optional), tags (optional)

Response 201:
{
  "file_id": "f_1a2b3c4d5e",
  "name": "document.pdf",
  "size": 1048576,
  "mime_type": "application/pdf",
  "folder_id": "folder_root",
  "created_at": "2024-10-15T14:23:45Z",
  "download_url": "https://cdn.cloudsync.io/files/f_1a2b3c4d5e",
  "checksum": "sha256:abc123..."
}

3.2 Download File
GET /files/{file_id}/download
Response: Binary file data with Content-Disposition header

3.3 List Files
GET /files?folder_id={id}&page={n}&limit={l}&sort={field}&order={asc|desc}

Parameters:
- folder_id: string (optional) - filter by folder
- page: integer (default: 1, max: 1000)
- limit: integer (default: 20, max: 100)
- sort: name|size|created_at|updated_at (default: created_at)
- order: asc|desc (default: desc)

6. ERROR CODES

Code | HTTP | Description
-----|------|-------------
1001 | 400  | Invalid file format - only PDF, DOC, DOCX, PNG, JPG allowed
1002 | 400  | File size exceeds 10GB limit
1003 | 401  | Authentication token expired or invalid
1004 | 403  | Insufficient permissions for this operation
1005 | 404  | File or folder not found
1006 | 409  | File already exists (use PUT to overwrite)
1007 | 413  | Request body too large
1008 | 429  | Rate limit exceeded - retry after {seconds}
1009 | 500  | Internal server error - contact support
1010 | 503  | Service temporarily unavailable

7. WEBHOOKS

POST https://your-server.com/webhook
Payload example (file.uploaded event):
{
  "event": "file.uploaded",
  "timestamp": "2024-10-15T14:23:45Z",
  "data": {
    "file_id": "f_1a2b3c4d5e",
    "name": "document.pdf",
    "size": 1048576
  }
}

Webhook security: Verify X-CloudSync-Signature header (HMAC-SHA256)
